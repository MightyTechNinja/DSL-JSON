GraphLogic Rules Engine DSL Documentation

This documentation provides an overview and instructions for writing code using a Domain-Specific Language (DSL) to define rules in a rules engine. The DSL supports conditional logic for defining actions based on various properties of entities such as Person, Skill, and Users.

Overview

The DSL allows the creation of complex rules with nested conditions, logical operators (AND, OR, NOT), and hierarchical relationships. The rules are written in a structured format using IF, THEN, ELSE, and ENDIF statements.

Basic Structure

A rule is defined using the following structure:

IF (condition) THEN action ENDIF

Nested Conditions

Rules can have nested conditions for more complex logic:

IF (condition) THEN
    IF (nested_condition) THEN nested_action ENDIF
ELSE
    action_else
ENDIF

Logical Operators

The DSL supports logical operators to combine multiple conditions:

  • AND
  • OR
  • NOT

Relationships

The DSL can express relationships between entities using the - HAS {property} -> syntax.

Syntax

Conditions

Conditions check the properties of entities. The basic format is:

(Entity {property: value})

Examples:

(Person {first_name: "John"})
(Skill {name: "JavaScript"})

Actions

Actions are functions or operations executed when conditions are met. Actions are defined as:

action()

Examples:

doSomething()

Combining Conditions

Conditions can be combined using logical operators:

  • AND: Both conditions must be true.
  • OR: At least one condition must be true.
  • NOT: The condition must be false.

Examples:

(Person {first_name: "John"}) AND (Person {age: 30})
(Person {first_name: "John"}) OR (Person {age: 30})
NOT (Person {first_name: "John"})

Relationships

Relationships between entities can be expressed as follows:

(Entity {property: value}) - HAS {property: value} -> (Entity {property: value})

Examples:

(Person {first_name: "John"}) - HAS {years: 10} -> (Skill {name: "JavaScript"})

Examples

Simple Condition

IF (Person {first_name: "John"}) THEN doSomething() ENDIF

Condition with ELSE

IF (Person {first_name: "John"}) THEN doSomething() ELSE THEN doSomethingElse() ENDIF

Nested Conditions

IF (Person {first_name: "John"}) THEN
    IF (Person {age: 30}) THEN doSomething() ENDIF
ELSE
    doSomethingElse()
ENDIF

Condition with AND

IF (Person {first_name: "John"}) AND (Person {age: 30}) THEN doSomething() ENDIF

Condition with OR

IF (Person {first_name: "John"}) OR (Person {age: 30}) THEN doSomething() ENDIF

Condition with NOT

IF NOT (Person {first_name: "John"}) THEN doSomething() ENDIF

Complex Logical Expression

IF (Person {first_name: "John"}) AND (Person {age: 30}) OR NOT (Skill {name: "JavaScript"}) THEN doSomething() ENDIF

Relationship Condition

IF (Person {first_name: "John"}) - HAS {years: 10} -> (Skill {name: "JavaScript"}) THEN learnJavascript() ENDIF

Complex Nested Conditions

IF (Users {email: "kamruljpi@gmail.com"}) AND (Users {age: 30}) OR NOT (Roles {name: "Admin"}) THEN
    doSomething()
ELSE
    IF (Person {first_name: "John"}) AND (Person {age: 30}) OR NOT (Skill {name: "JavaScript"}) THEN doSomething() ENDIF
ENDIF

Conclusion

This documentation outlines the syntax and structure for writing rules using the provided DSL. The flexibility of the DSL allows for the creation of both simple and complex rules to meet various requirements in a rules engine. Use this guide as a reference for defining and implementing your rules accurately.
